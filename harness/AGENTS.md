This session was launched via `HARNESS=opencode workflow-role`, which set `CURRENT_ROLE=workflow` and started OpenCode in a project directory.

## How instructions reach you

1. **Role instructions** — injected by a system transform plugin. These tell you to read `.workflow/active-role.md` for your specific workflow assignment.
2. **Infrastructure manifest** — injected alongside role instructions.
3. **Project instructions** — loaded by OpenCode from AGENTS.md files and `.opencode/opencode.json`.
